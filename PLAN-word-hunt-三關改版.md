## 現況（依 Ivan 描述）

單一 HTML 檔（HTML/CSS/JS 同檔）的英文單字連連看：
- 上方顯示圖示對應 5 個英文單字
- 下方字母格子，玩家拖曳選字（橫/直/斜皆可）

## 目標

改成三關流程。**只動流程與計分顯示**，以下既有機制一律不改：拖曳選字邏輯、字母格子產生邏輯、閃紅回饋邏輯。維持單一檔案，不拆檔、不引入框架或外部資源。

## 需求細節

### 1. 關卡資料常數（方便 Ivan 之後自己改內容）

檔案頂部（script 開頭）定義預留常數，格式貼齊原檔的單字資料結構（原檔單字若含圖示欄位，LEVELS 內每個單字也要帶同樣欄位）：

```js
const LEVELS = [
  { words: [/* 3 個較短單字，含各自圖示 */] },  // 關卡一
  { words: [/* 4 個單字 */] },                  // 關卡二
  { words: [/* 5 個單字 */] },                  // 關卡三
];
```

先填上可玩的預設內容（關卡一 3 個短字、關卡二 4 個、關卡三 5 個），單字沿用原檔既有題庫拆分即可，Ivan 之後會自行替換。

### 2. 星級門檻常數（每關獨立）

```js
// 每關 [三星門檻秒數, 二星門檻秒數]，超過二星門檻給一星
const STAR_THRESHOLDS = [
  [20, 40],   // 關卡一
  [30, 60],   // 關卡二
  [45, 90],   // 關卡三（單字多，門檻較寬鬆）
];
```

實際數值依原檔現有星級門檻等比推算，做成常數即可，Ivan 會自行調整。

### 3. 關卡流程

- 每一關開始：用該關的 `LEVELS[i].words` **重新產生**字母格子與上方圖示（呼叫既有的產生函式，不重寫它），並**重新啟動計時器**（每關獨立計時，不累計）。
- 找完該關全部單字 → 停止該關計時 → 依 `STAR_THRESHOLDS[i]` 算出 1~3 星。
- 記錄每關結果：`{ seconds, stars }`，供總結畫面使用。

### 4. 過關畫面（每關結束顯示）

- 顯示：本關花費秒數 + 星級（用 ★★☆ 這種實心/空心星呈現）
- 按鈕：第一、二關顯示「下一關」；第三關顯示「完成遊戲」
- 樣式貼齊原檔既有 UI 風格（顏色、字體、按鈕樣式跟原檔一致）

### 5. 總結畫面（三關全部完成後）

- 列出每一關：秒數 + 星級（★★☆ 形式）
- 顯示三關星星總和（最高 9 顆）
- 「重新開始整個遊戲」按鈕 → 清空所有關卡紀錄，回到關卡一重新開始

### 6. 螢光筆划詞效果（視覺升級）

把拖曳選字的**視覺呈現**改成像螢光筆畫過一樣順滑的色帶（經典 Word Search App 的那種效果）。只換視覺層，**選字判定邏輯（橫/直/斜、起點終點吸附）完全不動**。

具體要求：
- 拖曳時，從起始格中心到目前手指/滑鼠位置畫一條**半透明、兩端圓頭的粗色帶**，粗細約等於一格的 70~80%，字母顯示在色帶上方（色帶墊在字母下層）。
- 色帶方向吸附到八方向（橫/直/斜），但**長度跟著指標連續變化**——手指滑到兩格之間時色帶也跟著平滑伸長，不是一格一格跳，這是「順」的關鍵。
- 選對：色帶定格保留在該單字上，每個找到的單字用不同顏色（準備 5~6 色的循環色盤，半透明）。
- 選錯：沿用既有閃紅回饋——色帶閃紅後淡出消失。
- 實作建議：在格子上疊一層 SVG，用 `<line>` 加 `stroke-linecap="round"` + 粗 `stroke-width` 畫色帶，跟著 pointermove 更新端點即可，最簡單也最順；用 canvas 疊層或 CSS 圓角 div 旋轉也可，擇一貼齊原檔風格。
- 觸控：格子區設 `touch-action: none`，用 pointer events 統一滑鼠與觸控，拖曳中不可觸發頁面捲動。
- 效能：更新色帶端點用 `requestAnimationFrame` 或直接改 SVG 屬性，不可每次 pointermove 重建 DOM。

### 7. 不可動的部分（紅線）

- 拖曳選字**判定邏輯**（橫/直/斜判定、格子命中、選取結果）——第 6 點只換視覺層，判定照舊
- 格子產生邏輯（只改「餵進去的單字組」，不改產生演算法）
- 閃紅回饋邏輯（第 6 點的色帶閃紅是沿用它，不是重寫）
- 若原檔有外部單字注入鏈（`postMessage` / `#lessonData` hash / `apply*Data`），**不要弄斷**；注入資料與三關的整合方式先回報 Ivan 確認再動（例如外部資料進來時如何分配到三關）。若原檔沒有注入鏈，不需要新增，維持現狀即可。
